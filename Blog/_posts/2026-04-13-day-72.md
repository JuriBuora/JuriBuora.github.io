# 🔄 Topic

Recognizing the shift from using commands to understanding systems.

---

# 🎯 Goal

Reflect on how individual troubleshooting steps connect into a broader investigative mindset.

---

# 🛠 What I Did

Over the past sessions, I worked on:

- SSH troubleshooting  
- network connectivity testing  
- port verification  
- VM networking configuration  

These were initially isolated problems.

---

# 🔗 Key Cybersecurity Connections

In real SOC work, issues are rarely isolated.

Instead, analysts must:

- correlate signals  
- understand systems  
- connect multiple data points  

---

## 🔍 Investigation Questions

- What layer is failing?
- Is this network, service, or configuration?
- What evidence supports each hypothesis?

---

## 🚨 Detection Opportunities

- correlation across logs  
- identifying patterns instead of single events  
- combining network + endpoint signals  

---

## 🧭 MITRE ATT&CK Techniques

- T1046 — Network Service Discovery  
- T1021 — Remote Services  

---

# ⚠ Challenges

Moving from:

- command execution  
→ to  
- system understanding  

---

# 📚 What I Learned

- tools are only part of the process  
- investigation requires context  
- systems thinking is essential  

---

# ➡ Next Steps

- begin writing real detection rules  
- analyze logs from lab activity  

---

# 🧠 Reflection

This feels like a transition point:

> less “running commands”  
> more “understanding what’s happening”

---

# 🧩 Lessons Learned

**What worked**

Breaking problems into layers.

**What broke**

Seeing each issue as isolated.

**Why it broke**

Lack of system-level perspective.

**Fix / takeaway**

Always connect signals into a bigger picture.

---

# 📈 Skill Progression Context

This marks the shift toward real SOC-level thinking and investigation.
